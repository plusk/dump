Our client is the imaginary pizza maker "Pizzaperfeksjonisten". As such we do not have a specific contact person. Here is our website.

The changes we have made

Filename Changes
template (general page structure on all html pages) Some changes were made to the layout of the site. The footer no longer redirects to a separate sitemap page, as the sitemap is nowhere near elaborate enough to warrant its own page. The sitemap is in the footer itself. We also added social media links to the footer, and our logo is on the header of the page. The header text and the logos link to index.html for easy navigation. The footer even features a "back to top" link!
index.html Previously known as home.html, the index no longer feature star ratings on the reviews. The reviews are now random on page load, and the index slideshow starts on a random image, then loads in order and loops the slideshow at the end. The height of the header image automatically adjusts to the header image, as the slideshow images and the random headers for non-index pages have a different height.
menu.html Made as specified by the design document, the text is now orange(#FF9736) as opposed to black, following our color scheme.
classic_pizza.html
premium_pizza.html		 The layout of these pages was changed/improved significantly. The information related to each pizza is presented in an orderly fashion and removed the use of menuAnimation.js, which we will get to later. Premium pizza was previously known as special pizza.
allergy_list.html The table roughly follows the mockup, and naturally it has been expanded to account for our entire menu and all possible allergens. There is now also a symbol for pizzas that have traces of an allergen, so that either an allergen is present, not present, or traces of it may be present.
ordering.html Ordering is one of the larger pages on our site. This page has been expanded with some new input fields, the fields are now categorized and neatly presented. As opposed to writing the orders to a server-side textfile, the page now posts the forms via email to the client. This method was suggested by the project roadmap, so we changed the page to use it. The page also warns you if one of the required fields are empty, and the method of selecting how many and which pizzas you want has been reworked.
about_us.html This page remains mostly unchanged and serves as the backstory for the fictitious company, Pizzaperfeksjonisten. The group-photo of the creators has been removed, as no such photo exists.
restaurant.html Mostly unchanged, slideshow moved down, numbers integrated in text.
contact.html Mostly unchanged, googlemaps functionality is provided by javascript file.
metabout.html New html file that serves as a description of the site itself, the sources of the images and so on.
main.css The css is largely similar to what was shown in the mockups, as we actually made the core css and then took pictures of it for the mockups. The file contains all the css for the site and as such the file has become longer over time.
fixedNavbar.js This file was originally its own javaScript file, but we remade it using mostly jQuery, as a part of template.js. Our new solution is less jumpy/choppy, and it executes after the header has been loaded in with jQuery.
googlemaps.js A separate JS file for the Google Maps functionality on the contact page.
ordering.js As detailed in the ordering.html changes, the ordering works slightly different from the original thought. We now send the form data by email, as opposed to writing to a server side text file.
randomHeader.js New javascript file that loads up a random header image on page load. This script is connected to all of the pages, except for index, which has its own index slideshow.
randomReview.js Simple javascript file that puts a random review in the slideshow text box on index.file. Random review for every page load.
rollover.js Working as intended, hovering over a menu reveals a drop down menu of all the sub-pages.
randomPizza.js New file that adds some flavor to menu.html. A random classic pizza is features in the link to classic pizzas, and the same applies to the premium box.
slideshowIndex.js
slideshowRestaurant.js		 These files work as intended, minor change in work division (detailed further down).
menuAnimation.js This was scrapped as it didn't add anything meaningful to the menu, enough information was already on display. We believe we have enough JavaScript to fulfill the requirements, so we figured it was safe to drop a redundant function.
konami.js New file, dumb little easter egg code. Doesn't really do anything meaningful for the site, mostly added for the fun of it. Redirects to a different site when you perform the konami code.
Modularity template.js is our attempt at modularity. In this javascript file we utilize jQuery to access two SSI documents, namely header.part and footer.part. The header and footer are the same on every page, so we figured it'd make sense to load it as a template, so that any changes to those files would apply to all our html pages. After the header is loaded up, a callback function goes into the nav and highlights the current page in the navbar. It's all very dynamic and fun. Most of our JavaScript applies to all our pages, so the code is in external files.
